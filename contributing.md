# Contributing Guidelines

Thank you for your interest in contributing to Awesome QGIS! Please make sure your pull request follows these guidelines:

## Adding a Resource

- Search through the previous pull requests before making a new one.
- Use the following format: `- [Resource Title](url) - Description ending with a period.`
- The description should not start with "A" or "An".
- Keep descriptions short, clear, and informative.
- Expand on why the resource is useful in your pull request if needed.
- Any added projects should be at least 30 days old.
- Only add resources you can personally recommend.

## General

- Add new categories or improve existing categories if possible.
- Double check your spelling and grammar.
- Make sure your text editor is set to remove trailing whitespace.
- Run [awesome-lint](https://github.com/sindresorhus/awesome-lint) locally before submitting: `npx awesome-lint`
